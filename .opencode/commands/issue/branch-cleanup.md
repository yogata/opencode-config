# ブランチクリーンアップ

mainブランチにマージ済みのリモートブランチ、ローカルブランチ、worktreeを一括削除します。

## 手順

| ステップ | 操作                     | コマンド例                                          |
| -------- | ------------------------ | --------------------------------------------------- |
| 1        | 現在の状態確認           | `git worktree list`, `git branch --merged main`     |
| 2        | 削除対象の特定・提示     | パターン: `feature/*`, `fix/*`, `bugfix/*`          |
| 3        | ユーザー確認             | "続行しますか？ (y/n)"                              |
| 4        | Worktree削除             | `git worktree remove <path>`                        |
| 5        | mainブランチに切り替え   | `git checkout main && git pull`                     |
| 6        | ローカルブランチ削除     | `git branch -d <branch>`                            |
| 7        | リモートブランチ削除     | `git push origin --delete <branch>`                 |
| 8        | Prune実行                | `git fetch --prune`                                 |
| 9        | 結果報告                 | 削除数・ブランチ名一覧                              |

## 除外条件

- `main`, `master`, `develop`, `staging` は削除しない
- 未マージのブランチは削除しない
- ユーザーが明示的に残したいと指定したブランチは削除しない

## オプション

| オプション                 | 説明                   |
| -------------------------- | ---------------------- |
| `--dry-run`                | 削除対象を表示のみ     |
| `--local-only`             | ローカルブランチのみ   |
| `--remote-only`            | リモートブランチのみ   |
| `--include-pattern <pat>`  | 特定パターンのみ対象   |

## エラーハンドリング

| エラー                     | 対処                             |
| -------------------------- | -------------------------------- |
| worktree内に未コミット変更 | 警告表示、スキップするか確認     |
| ローカルブランチ削除失敗   | スキップして次へ                 |
| リモートブランチ削除失敗   | エラー表示、ローカルのみ削除継続 |
| mainにいない場合           | 自動的にmainに切り替え           |
