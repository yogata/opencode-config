## NG理由

以下の分類から該当するものを選択：

- [ ] 仕様バグ: 要件定義と実装の間に論理的矛盾がある場合
- [ ] 実装バグ: 要件定義は正しいが実装が仕様を満たさない場合
- [ ] スコープ外逸脱: 実装が要件定義の範囲を超えている場合
- [ ] テスト不足: テストカバレッジが品質基準を満たさない場合
- [ ] 品質基準未達: コード品質がプロジェクト基準を満たさない場合

## 修正方針

### REQ更新要否

- 仕様バグ → REQ UPDATE（要件修正）
- 実装バグ → コード修正（REQ変更不要）
- スコープ外逸脱 → REQ UPDATE（スコープ明確化）
- テスト不足 → テスト追加（REQ変更不要）
- 品質基準未達 → リファクタリング（REQ変更不要）
