# 学びの昇華判定とスタブ生成

`docs/tips/evaluation-report.md` のクラスタと `docs/tips/archive.md` の過去エントリから昇華判定を行い、ユーザー承認後に `docs/tips/elevation-staging/` にスタブファイルを生成する。

`.opencode/` への直接配置は行わない。staging領域のファイルをユーザーが確認後、手動で適切な場所に移動する。

## Steps

1. **evaluation-report.mdの存在確認**:
   - `docs/tips/evaluation-report.md` を確認
   - 存在しない → エラー終了。「先に `/tips-refactor` を実行して分析レポートを生成してください」

2. **データ読込**:
   - evaluation-report.md を読込（クラスタ一覧・重み・エントリ数を把握）
   - archive.md を読込（該当クラスタの過去エントリを確認）

3. **昇華判定**（LLMが各クラスタを判定）:
   判定基準4段階:
   - **スキル化**: 汎用的なパターン、複数プロジェクト/コンテキストで再利用可能、3回以上の出現
   - **コマンド化**: 特定の操作フローが繰り返し現れている、自動化すべき、2回以上の出現
   - **ワークフロー組み込み**: 既存スキル/コマンド/ワークフローに追加すべき知見、1回以上で他システムとの関連あり
   - **保留**: まだ昇華の余地がない、情報が断片的、出現回数が少ない

4. **昇華推奨をユーザーに提示**:
   ```
   ## 昇華推奨

   | クラスタ | テーマ | 推奨 | 理由 |
   |---------|--------|------|------|
   | 1 | Windows環境エスケープ問題 | スキル化 | 3回出現、汎用的なパターン |
   | 2 | Supabase RLS落とし穴 | 保留 | 2回出現だが情報が断片的 |
   ```

5. **ユーザー承認**:
   - ユーザーが各クラスタの昇華判定を確認・修正
   - 承認したクラスタのみ処理
   - 承認しない → 「昇華をキャンセルしました」と報告して終了

6. **スタブ生成**（staging領域のみ）:
   - `docs/tips/elevation-staging/` に生成

   - スキル化推奨 → `{name}-skill-stub.md`:
     ```markdown
     # {name} スキルスタブ

     ## 動機
     以下の学びから昇華:
     - YYYY-MM-DD: タイトル
     - YYYY-MM-DD: タイトル

     ## Description（草案）
     {1-2行のスキル概要}

     ## TODO
     - [ ] 完全なSKILL.mdの作成
     - [ ] トリガー条件の定義
     - [ ] ステップの具体化
     - [ ] Guardrailsの定義
     ```

   - コマンド化推奨 → `{name}-command-stub.md`:
     ```markdown
     # {name} コマンドスタブ

     ## 動機
     以下の学びから昇華:
     - YYYY-MM-DD: タイトル
     - YYYY-MM-DD: タイトル

     ## Description（草案）
     {1-2行のコマンド概要}

     ## TODO
     - [ ] 完全なコマンドファイルの作成
     - [ ] 引数・オプションの定義
     - [ ] エラーハンドリングの定義
     - [ ] フロントマターの確定
     ```

   - ワークフロー組み込み → `{target-workflow}-integration-note.md`:
     ```markdown
     # {target-workflow} 統合ノート

     ## 組み込み先
     - {対象スキル/コマンド/ワークフロー}

     ## 追加すべき知見
     - {統合すべき内容}

     ## TODO
     - [ ] 対象スキル/コマンドに知見を反映
     - [ ] 動作確認
     ```

7. **完了報告**:
   - 生成したスタブファイル一覧
   - 最終確認: 「.opencode/ への配置は別途手動で行ってください。staging領域のファイルを確認後、適切な場所に移動してください」

## Error Handling

| エラー | 対処 |
|--------|------|
| evaluation-report.mdが存在しない | エラー終了。「先に `/tips-refactor` を実行してください」 |
| クラスタが0件 | 「昇華対象のクラスタがありません」と報告して終了 |
| ユーザーが承認しない | 「昇華をキャンセルしました」と報告して終了 |
| staging領域の書込失敗 | エラー内容を報告 |

## 使用例

```
/tips-elevate
→ evaluation-report.md + archive.md を読込
→ 各クラスタの昇華判定を実行（スキル化/コマンド化/ワークフロー組み込み/保留）
→ 判定結果をテーブルで提示し、ユーザー承認を求める
→ 承認後、staging領域にスタブを生成
→ 完了報告
```

## 注意事項

- **staging領域のみに生成**: `.opencode/skills/` や `.opencode/commands/` への直接配置は禁止
- **スタブのみ生成**: 完全なSKILL.mdやコマンドファイルは生成しない（TODOセクション付きのスタブ）
- **evaluation-report.mdとarchive.mdは読込専用**: 変更・削除は禁止
- **ユーザー承認必須**: 承認なしにファイルを生成しない
- **セマンティック分析**: エントリの分類と品質評価にLLMによる分析を使用
