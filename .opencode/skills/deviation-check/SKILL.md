# Deviation Check スキル

このスキルは、実装と要件の乖離を検出し、品質ゲートとして機能する。

---

## 概要

実装と要件の乖離検出・品質ゲート。②構造的実行フェーズの完了時と③レビュー完了フェーズで使用。

- **役割**: 要件ドキュメント（HLLD/LLD）と実際の実装を比較し、乖離を検出
- **タイミング**: ②構造的実行フェーズの完了時（PR作成前）と③レビュー完了フェーズ（マージ前）
- **依存**: issue-guide スキルから参照される専門スキル

---

## 乖離の定義

何をもって「乖離」とするかの基準を定義する。

### 重大乖離

要件docに記載された受け入れ基準を満たさない実装。

- **例**: チェックボックスが未実装
- **例**: 受け入れ基準のGiven-When-Thenを満たさない
- **例**: 要件に明記された機能が欠落している

### 軽微乖離

実装は受け入れ基準を満たすが、意図しない追加変更を含む。

- **例**: スコープ外ファイルの変更
- **例**: 要件にないリファクタリング
- **例**: 追加された設定変更

---

## 検出観点

エージェントが乖離を検出する際の観点を定義する。

### 要件docのチェックボックス vs 実装の対応

- 全ボックスが実装されているか
- 各ボックスの内容が正しく実装されているか

### work planの範囲 vs 実際の変更ファイル

- 計画外の変更がないか
- 変更ファイルがwork planの範囲内か

### 決定事項の Consequences vs 実際の影響

- 予想外の副作用がないか
- 決定事項の影響範囲と実際の影響が一致しているか

---

## 報告フォーマット

乖離の報告形式を定義する。

### 報告項目

- **影響度**: 重大 / 軽微
- **対象**: 要件docの該当セクション / 変更ファイル
- **内容**: 乖離の具体的な説明
- **推奨アクション**: 修正 / 承認 / 差し戻し
- **理由**: 推奨アクションの根拠

### 報告例

```
## 乖離検出報告

### 乖離1
- **影響度**: 重大
- **対象**: 要件doc セクション3.2 / src/components/Button.tsx
- **内容**: チェックボックス「バリデーションエラー表示」が未実装
- **推奨アクション**: 修正
- **理由**: 受け入れ基準を満たしていない

### 乖離2
- **影響度**: 軽微
- **対象**: 変更ファイル / src/utils/format.ts
- **内容**: スコープ外のユーティリティ関数がリファクタリングされている
- **推奨アクション**: 承認
- **理由**: 品質向上であり、要件を満たしている
```

---

## ループバック判定

①バイブス壁打ちへの差し戻し基準を定義する。

### 差し戻し基準

- **重大乖離が2件以上** → ①バイブス壁打ちフェーズ全体を差し戻し
- **重大乖離1件** → 該当する要件docのセクションのみ再壁打ち
- **軽微乖離のみ** → そのまま進行（乖離内容を決定事項として decision-log に記録）

### 重要事項

自動ループバックはしない。エージェントが推奨アクションを提案し、ユーザーが決定する。
