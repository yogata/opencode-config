# Review Workflow - 運用ルール変更時のレビュー手順

## 目的
Obsidian Vaultの運用ルールが変更された場合、その変更が既存スキルに与える影響を評価し、必要な更新を行うための手順。

## 手順

### 1. 変更の特定
- [ ] どの運用ルールが変更されたかを確認 (`AGENTS.md`, `40_resources/AGENTS.md`)
- [ ] 変更の内容を文書化（追加・変更・削除されたルール）

### 2. 影響分析
- [ ] `skill-inventory.md` を参照し、全スキルを確認
- [ ] 各スキルの「References」セクションをチェック
- [ ] 変更されたルールを参照しているスキルを特定
- [ ] 影響レベルを分類：
  - **高**: 核心機能・前提条件が変更
  - **中**: 詳細動作・推奨事項が変更
  - **低**: 補足情報・例が変更

### 3. 優先順位付け
- [ ] 影響レベルに基づき更新順序を決定
- [ ] 高 → 中 → 低 の順で更新

### 4. スキル更新
各影響スキルに対して：
- [ ] 該当箇所を特定
- [ ] 新ルールに従い内容を更新
- [ ] 変更内容を簡易ログに記録

### 5. 検証
- [ ] 更新されたスキルが新しい運用ルールと整合しているか確認
- [ ] 必要に応じテストケースを作成

### 6. 文書化
- [ ] 変更履歴を更新
- [ ] スキル間の参照関係が正しいことを確認

## 出力物
- 更新されたスキルファイル群
- レビューログ（別ファイルまたは追記）
