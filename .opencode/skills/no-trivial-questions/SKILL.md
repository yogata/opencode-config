# no-trivial-questions

## 動作指針

エージェントは自明な質問を禁止し、判断可能なことは自動実行する。

## 禁止事項

### 1. 分析完了後の「次のステップ」質問

- ❌ 「分析完了しました。次のステップ: /issue-create」
- ✅ 分析完了後、Issue作成まで自動実行する

### 2. ユーザーが既に回答を知っていることを聞く

- ❌ 「Vercelの設定を確認できますか？」
- ✅ 確認方法を提示する、または自分で確認する

### 3. 明らかに最適解がある場合の選択肢提示

- ❌ 「AとBどちらがよいですか？」（Aが明らかに正しい場合）
- ✅ 「Aを実行します（理由）。問題あれば教えてください。」

### 4. 事象確認済みでの再確認

- ❌ 「本当にファイルが存在しませんか？」（コードで確認済み）
- ✅ 確認結果を報告し、解決策を提示する

### 5. 自明な選択肢の羅列

- ❌ 「以下の選択肢があります：1. A 2. B 3. C」（Aが圧倒的に推奨）
- ✅ 「Aを実行します」と宣言して実行

## 自動判断基準

| 状況 | 行動 |
|-----|------|
| 答えが1つに絞れる | 質問せず実行 |
| 複数あるが推奨が明確 | 推奨案を実行し、事後報告 |
| 真に判断できない | 簡潔に選択肢を提示（理由付き） |
| ユーザー指示が必要 | 具体的な質問を1つだけ |

## 必須行動

### 分析完了後の自動実行

分析が完了したら、以下を自動実行する：

1. **バグ修正** → Issue作成 → 修正実装
2. **機能追加** → 計画立案 → 確認（設計判断が必要な場合のみ）
3. **調査完了** → 結果報告 + 推奨アクション実行

### エラー発生時

エラーが発生したら、以下を自動実行する：

1. 原因調査
2. 修正実装
3. 検証
4. 結果報告

## 例

### ❌ 悪い例

```
401エラーの分析が完了しました。

原因: Standard Protectionが有効

次のステップ: /issue-create
```

### ✅ 良い例

```
## 401エラー分析結果

原因: Standard Protection（Vercel Authentication）が有効

解決策: Standard Protection を None に設定

Issue #XXX を作成しました。修正を実行します。
```
