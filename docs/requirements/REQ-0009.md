> **⚠️ 非推奨**: この要件ファイルはREQ-0027（要件管理体制の再構築）により廃止されました。新たな領域別要件ファイルに統合されています。

## 背景・課題

現行のREQファイルフォーマットは frontmatter に多数のメタデータフィールド（classification, category, issue, adr, related_to, depends_on）を保持しているが、運用上これらのフィールドは実質的に活用されていない。README.mdのインデックステーブルも6列構成で情報密度が低く、メンテナンスコストに見合わない。

opencode-configレポジトリの要件フォーマットでは frontmatter 6フィールド（id, title, status, created, updated, tags）のみで十分に管理できている。

## 目標

REQフォーマットを opencode-config のシンプルな構成に統一し、メンテナンスコストを削減する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0009-01 | REQファイルの frontmatter から classification, category, issue, adr, related_to, depends_on を除去し、id, title, status, created, updated, tags の6フィールドのみにする |
| REQ-0009-02 | README.mdのインデックステーブルを ID, タイトル, ステータス の3列に簡素化する |
| REQ-0009-03 | README.mdから「分類」「カテゴリ」セクションを除去する |
| REQ-0009-04 | doc_requirement.md テンプレートの frontmatter を6フィールドに更新する |
| REQ-0009-05 | 全35個の既存REQファイル（REQ-0001〜REQ-0039）の frontmatter から不要フィールドを除去する |
| REQ-0009-06 | validation.test.ts から classification 関連の検証（REQUIRED_FIELDS, VALID_CLASSIFICATIONS, テスト5, テスト6）を除去する |
| REQ-0009-07 | e2e-scenario.test.ts から classification, related_to, depends_on 関連の検証（VALID_CLASSIFICATIONS, シナリオ1, シナリオ2, シナリオ7）を除去・修正する |

## 非機能要件

- 既存REQファイルの本文（背景・要件・スコープ等）は変更しない
- frontmatterの変更によりテストが通らなくなることを防ぐため、テストファイルの修正を同期させる
- ADRは不要（フォーマット変更のみで、アーキテクチャ判断を含まない）

## スコープ

- **対象**: docs/requirements/ 配下のREQファイル、README.md、doc_requirement.md テンプレート、REQバリデーションテスト
- **対象外**: REQファイルの本文内容の変更、REQファイル間の依存関係の解決、仕様書（docs/specs/）の更新
