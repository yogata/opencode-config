## 目的

Issue・PR・REQ・ADR・コメント等の各種アーティファクト生成に使用するテンプレート体系を定義する。テンプレートの命名・配置・変数規約を統一し、コマンド間で一貫した出力形式を担保するための要件である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0009-001 | テンプレートファイルは責任範囲に基づいて配置するものとする。Issue/コメント系テンプレートは`.opencode/skills/issue-template-manager/templates/`に、REQテンプレートは`.opencode/skills/req-file-manager/templates/`に、ADRテンプレートは`.opencode/skills/adr-file-manager/templates/`に、乖離検出レポートテンプレートは`.opencode/skills/spec-compliance/templates/`に、PR説明テンプレートは`.opencode/commands/issue/templates/`に配置し、ファイル種別に応じたプレフィクス（`issue_desc_`・`issue_comment_`・`doc_`・`pr_`・`report_`）で命名するものとする（SHALL） |
| REQ-0009-002 | Issue作成テンプレートはバグ修正（`issue_desc_bug.md`）と機能追加・変更（`issue_desc_feature.md`）で分離し、Epic用（`issue_desc_epic.md`・`issue_desc_child.md`）を独立して提供するものとする。各テンプレートは対応するPattern（A: bug, B: enhancement/feature, C: refactor/maintenance, D: docs/chore）のラベルをサポートするものとする（SHALL） |
| REQ-0009-003 | ドキュメントテンプレート（`doc_requirement.md`・`doc_adr.md`）はfrontmatterとセクション構成のひな形を提供し、変数プレースホルダー（`{NNNN}`・`YYYY-MM-DD`等）を含むものとする（SHALL） |
| REQ-0009-004 | REQテンプレートは目的・要件・適用範囲を必須セクションとし、Acceptance Criteria・Verification Method・Change Historyセクションを追加してもよい。要件テーブルにはRFC 2119言語の使用例を含めるものとする（SHALL/MAY） |
| REQ-0009-005 | PR説明テンプレート（`pr_desc.md`）と乖離検出レポートテンプレート（`report_spec_compliance.md`）は実行結果の記録用フォーマットとして提供するものとする。乖離検出レポートは可変長形式とし、0件・1件・3件以上のいずれにも対応しなければならない（SHALL） |
| REQ-0009-006 | バックログ用テンプレート（`issue_desc_backlog_epic.md`・`issue_desc_backlog_child.md`）はissue-backlogコマンド専用に提供するものとする（SHALL） |
| REQ-0009-007 | テンプレートファイルのセクション見出しは日本語で記述するものとする（SHALL） |
| REQ-0009-008 | Issue本文テンプレート（`issue_desc_feature`・`issue_desc_bug`・`issue_desc_epic`・`issue_desc_child`・`issue_desc_backlog_epic`・`issue_desc_backlog_child`）に受け入れ条件セクションを必須セクションとして追加するものとする（SHALL） |
| REQ-0009-009 | PR本文テンプレート（`pr_desc.md`）に受け入れ条件セクションを必須セクションとして追加するものとする（SHALL） |
| REQ-0009-010 | 受け入れ条件セクションはGiven-When-Then形式の完了判定基準を記述するものとし、テスト戦略セクションとは独立セクションとして共存するものとする（SHALL） |

## 適用範囲

- **対象**: テンプレートファイルの配置場所・命名規則・プレースホルダー規約、各テンプレートのセクション構成
- **対象外**: テンプレート内で使用する変数の解決ロジック、テンプレートの動的生成、コマンド定義内でのテンプレート参照手順
