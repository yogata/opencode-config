## 背景・課題

issue-close Step 7 は親Epic本文のステータストラッキング表を更新するが、全子Issue完了時のEpic自体の自動クローズを行わない。Issue #82（Epic）が子Issue #84,#85,#86,#87 全完了後もOPENのまま残り、手動クローズが必要だった。

## 目標

issue-close実行時に、同一Epic内の全子Issueが完了していればEpicも自動的にクローズする。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0018-01 | issue-close Step 7 で親Epic本文更新後、Epic内の全子Issue状態を `gh issue view --json state` で確認する |
| REQ-0018-02 | 全子IssueがCLOSEDの場合、Epicに完了コメントを追記し `gh issue close --reason completed` でクローズする |
| REQ-0018-03 | 1件以上OPEN子Issueが残っている場合、スキップして完了報告に「Epic #{N}: N件未完了のためスキップ」と表示する |
| REQ-0018-04 | 親Epicが存在しない（独立Issue）場合、従来通りスキップする |
| REQ-0018-05 | 子Issue状態取得失敗時は警告表示してスキップ（Epicクローズしない） |
| REQ-0018-06 | 完了報告にEpicクローズ/スキップ状況を含める |

## 非機能要件

- 子Issue状態確認は `gh issue view --json state` で高速に実行
- `gh-cli-best-practices` に従いコメント追記は `--body-file` 使用

## スコープ

- **対象**: `issue-close.md` Step 7 の拡張（1ファイル）
- **対象外**: issue-guide-reports のフォーマット変更、他コマンドへの影響
