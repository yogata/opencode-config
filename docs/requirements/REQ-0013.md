## 目的

issue-backlog コマンドを承認前処理（抽出・分類・確認）と承認後処理（Issue作成）に分割し、抽出候補の解消済み確認機能を追加することで、バックログ抽出の安全性・運用性・ユーザー制御性を向上させる。

## 要件

### コマンド分割

| ID | 要件 |
|---|---|
| REQ-0013-001 | `issue-backlog` を承認前処理コマンドとし、残課題の抽出・分類・解消済み確認・確認用レポート生成までを担当するものとする（SHALL） |
| REQ-0013-002 | `/issue/issue-backlog` は GitHub Issue/PR を作成してはならない（SHALL） |
| REQ-0013-003 | `/issue/issue-backlog` は抽出元 issue/PR に `backlog-extracted` コメントを投稿してはならない（SHALL） |
| REQ-0013-004 | `/issue/issue-backlog` は確認結果を `.sisyphus/drafts/backlog-draft-{period-slug}.md` に保存するものとする（SHALL） |
| REQ-0013-005 | 承認後コマンド `/issue/issue-backlog-create` を新設するものとする（SHALL） |
| REQ-0013-006 | `/issue/issue-backlog-create` は承認済み backlog draft を入力として Epic Issue と子Issueを作成するものとする（SHALL） |
| REQ-0013-007 | `/issue/issue-backlog-create` は Issue 作成完了後、抽出元 issue/PR に `backlog-extracted` コメントを投稿するものとする（SHALL） |
| REQ-0013-008 | `backlog-extracted` コメントの付与は Epic + 子Issue 作成完了後に限定するものとする（SHALL） |

### Draft 構造・ライフサイクル

| ID | 要件 |
|---|---|
| REQ-0013-009 | draft には対象期間・抽出元 issue/PR・抽出残課題・分類・優先度・元テキスト・ユーザー調整結果・解消済み確認結果・status を保存するものとする（SHALL） |
| REQ-0013-010 | draft status は `draft` → `approved` → `issued` の遷移とする（SHALL） |
| REQ-0013-011 | `/issue/issue-backlog` はユーザー承認時に draft status を `approved` として保存するものとする（SHALL） |
| REQ-0013-012 | 差し戻し中の draft status は `draft` のままとする（SHALL） |
| REQ-0013-013 | ユーザーによる draft ファイル直接編集は標準フローに含めない（SHALL） |
| REQ-0013-014 | `/issue/issue-backlog-create` は `approved` 状態の draft のみ処理するものとする（SHALL） |
| REQ-0013-015 | `/issue/issue-backlog-create` は引数なしの場合、`status: approved` の draft が1件のみなら自動検出して処理するものとする（SHALL） |
| REQ-0013-016 | `status: approved` の draft が複数ある場合、対象候補を表示して停止するものとする（SHALL） |
| REQ-0013-017 | `/issue/issue-backlog-create {period-slug}` による明示指定を許可するものとする（SHALL） |
| REQ-0013-018 | `draft` または `issued` 状態の draft は `/issue/issue-backlog-create` の処理対象外とし、エラーメッセージを表示して停止するものとする（SHALL） |

### 解消済み確認

| ID | 要件 |
|---|---|
| REQ-0013-019 | 抽出候補ごとに、抽出元 issue/PR のクローズ日より後、かつ同一対象期間内にクローズされた issue/PR を照合し、後続対応により解消済みでないかを確認するものとする（SHALL） |
| REQ-0013-020 | 解消済み確認は issue/PR のタイトル・本文・コメントを対象とする（SHALL） |
| REQ-0013-021 | commit は初期仕様の照合対象外とする（SHALL） |
| REQ-0013-022 | 解消済み確認の判定は `未解消` / `解消済み` / `不明` の3値とする（SHALL） |
| REQ-0013-023 | `解消済み` と判定された抽出候補は backlog issue 作成対象から除外し、レポート上で「除外: 修正済み」として根拠 issue/PR を示すものとする（SHALL） |
| REQ-0013-024 | `不明` と判定された抽出候補は backlog issue 作成対象に含め、draft に確認不明である旨を記録するものとする（SHALL） |
