## 背景・課題

`.sisyphus/` 配下の notepads と evidence ディレクトリのファイルが、archive-completed-plan スクリプト実行後に取り残されることがある。

**根本原因**: `archive_plan.py` は notepads で完全一致マッチング（`.sisyphus/notepads/<plan_name>/`）、evidence でプレフィクス glob マッチング（`<plan_name>`, `<plan_name>.*`, `<plan_name>-*`）を使用する。しかし、notepad ディレクトリ名が plan 名と完全に一致していない場合（例: plan `install-continuous-learning-v2-windows` に対して notepad `install-continuous-learning-v2/`）、ファイルがマッチせず取り残される。

**現状の orphaned notepads**:
- `adr-file-manager/` — SKILL作成時の学び（64行、有用）
- `git-init/` — 初期設定記録（22行、古い）
- `install-continuous-learning-v2/` — Claude Code 用 WSL 設定記録（241行、現在の環境と無関係）

## 目標

`.sisyphus/` 配下の全ファイルの命名規則を plan 名と統一し、archive-completed-plan で確実にアーカイブされるようにする。

## 機能要件

| ID | 要件 | 受け入れ基準 |
|---|---|---|
| REQ-0013-01 | AGENTS.md に `.sisyphus/` 配下の7カテゴリ（plans, drafts, evidence, execution, notepads, tasks, reports）の命名規則を追加する | `Select-String -LiteralPath "AGENTS.md" -Pattern "Sisyphus.*命名規則"` が1件ヒットすること |
| REQ-0013-02 | 命名規則は notepads のディレクトリ名が plan 名と完全一致することを明記する | notepads のルール記述に `<plan_name>/` 形式が含まれること |
| REQ-0013-03 | 命名規則は evidence のファイル名が plan 名プレフィクスであることを明記する | evidence のルール記述に `<plan_name>.*` または `<plan_name>-*` 形式が含まれること |
| REQ-0013-04 | 正例・誤例を含める | ✅ 例: plan `issue-2` → notepads `issue-2/`、❌ 例: plan `install-v2-windows` に notepad `install-v2/` |
| REQ-0013-05 | orphaned notepad `adr-file-manager/` の内容を tips-capture で昇華後、ディレクトリを削除する | tips の保存先に adr-file-manager の学びが含まれること、`.sisyphus/notepads/adr-file-manager/` が存在しないこと |
| REQ-0013-06 | orphaned notepad `git-init/` を削除する | `.sisyphus/notepads/git-init/` が存在しないこと |
| REQ-0013-07 | orphaned notepad `install-continuous-learning-v2/` を削除する | `.sisyphus/notepads/install-continuous-learning-v2/` が存在しないこと |

## 非機能要件

- **保守性**: 命名規則は AGENTS.md に集約し、全スキル・コマンドから参照可能にする
- **言語**: 日本語（既存 AGENTS.md に準拠）
- **後方互換性**: 既存の archive_plan.py は変更しない

## スコープ

- **対象**:
  - AGENTS.md への命名規則セクション追加（`## 開発環境` の後、append-only）
  - orphaned notepads の処理（tips-capture + 削除）
- **対象外**:
  - `archive_plan.py` の変更
  - `.sisyphus/evidence/` ディレクトリの空作成
  - 過去のアーカイブ（`.sisyphus/archives/`）の再編成
  - バリデーションスクリプト・pre-commit hook の作成
  - 既存 AGENTS.md `## 開発環境` セクションの変更
