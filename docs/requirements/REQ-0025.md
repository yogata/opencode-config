## 背景・課題

`skill-authoring-best-practices` スキルが定義する品質基準（5評価軸・4レビュープロトコル）に基づき、全19スキルを評価した結果、12スキルで Frontmatter Check 不合格（`DO NOT USE FOR` 欠落）、2スキルで日英混在の Advisory Check 要改善が判明した。

最も深刻な問題は **Trigger Precision（トリガー精度）** で、12/19スキルが `DO NOT USE FOR` を欠落している。これはエージェントのスキル選択精度を低下させ、誤選択のリスクを高める。

## 目標

全19スキルを `skill-authoring-best-practices` の Frontmatter Check 合格（19/19）に引き上げる。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0025-01 | 12スキルの description に `DO NOT USE FOR:` を追加する |
| REQ-0025-02 | 12スキルの `Use when` 表記を `USE FOR:` 表記に統一する |
| REQ-0025-03 | obsidian-note-workflow の Stage 説明（Capture/Organize/Solidify/Archive）を日本語に統一する |
| REQ-0025-04 | obsidian-vault-review の Overview と When to Use セクションの重複を排除する |

### REQ-0025-02 詳細: USE FOR 表記統一

`Use when` を含む12スキルの description 文言を `USE FOR:` 形式に変換する。

**変換パターン**:
- Before: `...Use when creating notes, organizing inbox, adding tags, or archiving notes.`
- After: `...USE FOR: creating notes, organizing inbox, adding tags, or archiving notes. DO NOT USE FOR: ...`

### REQ-0025-03 詳細: 日英混在の統一

obsidian-note-workflow の Stage 説明を日本語に統一:

| 現在 | 変更後 |
|------|--------|
| ### Capture (収集) | ### Capture（収集） |
| Capture ideas, bookmarks... | アイデア、ブックマーク、一時ノートを `00_inbox` に `type/inbox` タグ付きでキャプチャする。フォーマット不要 — 生のキャプチャが最優先。後で整理ステージで処理する。 |
| ### Organize (整理) | ### Organize（整理） |
| Move notes from `00_inbox` to... | `00_inbox` から `40_resources/41_inbox` にノートを移動し、適切なタグ（`type/note`、ドメインタグ）を適用して説明的なタイトルを割り当てる。各ノートは1つの明確な目的を持つこと。 |
| ### Solidify (定着) | ### Solidify（定着） |
| Transform organized notes into... | 整理済みノートを `40_resources/42_notes` の `type/permanent` タグ付き恒久ノートに変換する。関連ノートへの双方向リンクを追加し、長期的価値のために内容を洗練させ、原子性（1ノート1アイデア）を確保する。 |
| ### Archive (アーカイブ) | ### Archive（アーカイブ） |
| Move completed or inactive notes... | 完了または非アクティブなノートを `status/archived` タグ付きで `50_archives` に移動する。ノートは検索可能だがアクティブなワークスペースから外れる。定期的にレビューして再活性化を検討する。 |

### REQ-0025-04 詳細: セクション重複排除

obsidian-vault-review の以下の重複を統合:

- **現状**: 「## Overview」と「## When to Use」がほぼ同じ内容を説明
- **変更**: 「## Overview」に統合し、「## When to Use」は削除

## 非機能要件

- 全19スキルの Frontmatter Check 合格率: 37% → 100%
- description 変更によるスクリプト・設定ファイルへの副作用なし（description はメタデータのみ）
- 各スキルの行数は現状維持または減少（description 変更は数行）

## スコープ

- **対象**: `.opencode/skills/` 配下の12スキルの frontmatter description、obsidian系2スキルの本文
- **対象外**: すでに合格している7スキル（epic-status-tracker, skill-authoring-best-practices, obsidian-skill-maintainer, issue-guide-review, issue-guide-phases, issue-guide-reports, gh-cli-best-practices）の変更、参照ファイル（references/）の変更
