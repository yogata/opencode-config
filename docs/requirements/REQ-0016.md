## 背景・課題

Windows PowerShell環境で `gh` CLI（GitHub CLI）のUTF-8出力をPowerShell変数に格納すると、Shift-JISエンコーディングで文字化け（mojibake）が発生する。Issue #60の改善提案13として登録済み。

**具体例**: `gh issue view` の日本語本文を `$body = gh issue view ...` で取得 → 変数内で日本語が化ける → `gh issue edit --body $body` で化けた状態がGitHubに反映される。

**現状のgh-cli-best-practices SKILL.md**: WRITE安全性（`--body-file`による書き込み）のみ対応済み。READ安全性（gh出力の読み取り）が未対応。

## 目標

gh CLI出力を安全に読み取る手順をgh-cli-best-practices SKILL.mdに追加し、全issue-*コマンドがこの手順を参照するよう更新する。これによりWindows環境での文字化けを完全に防止する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0016-01 | gh-cli-best-practices SKILL.mdに「安全な読み取り手順」セクションを追加する |
| REQ-0016-02 | 読み取り手順: gh出力を一時ファイル（`.sisyphus/tmp/gh-read-{timestamp}.md`）にリダイレクトし、Read toolで読み取る |
| REQ-0016-03 | 禁止事項に「PowerShell変数への直接格納（`$var = gh ...`）」「サブ式での直接使用（`$(gh ...)`）」を追加する |
| REQ-0016-04 | 全issue-*コマンドの定義ファイル（issue-create, issue-close, issue-work, issue-update, issue-next）にgh-cli-best-practices参照を追加・明記する |
| REQ-0016-05 | system.mdのgh-cli-best-practices説明に読み取り安全性の記述を追加する |

## 非機能要件

- **後方互換性**: 既存のWRITE手順（`--body-file`）は変更しない
- **一時ファイル管理**: 読み取り用一時ファイルはRead tool使用後に削除する
- **エンコーディング**: 一時ファイルはUTF-8 (BOMなし)、改行コードLFで保存する

## スコープ

- **対象**:
  - gh-cli-best-practices SKILL.mdの更新（READ安全性セクション追加）
  - issue-*コマンド定義ファイルへの参照追加（5ファイル）
  - system.mdの説明更新
- **対象外**:
  - gh CLI自体の設定変更（`.bashrc`や環境変数は触らない）
  - PowerShell以外のシェル対応（WSL, Git Bash等）
  - issue-req, issue-save-req（これらはgh出力を読み取らない）
  - テストコードの追加（ドキュメント変更のみ）
