> **⚠️ 非推奨**: この要件ファイルはREQ-0027（要件管理体制の再構築）により廃止されました。新たな領域別要件ファイルに統合されています。

## 背景・課題

issue-*コマンド（req/create/work/close）の完了報告フォーマットに以下の問題がある：

- **CLI引数の露出**: `--req`, `--review-ng` 等の内部引数がユーザー向け出力に含まれる
- **issue-workの分岐過多**: 4つのNG分岐パスにspec-compliance、issue-update等の内部処理が混在
- **内部詳細の漏出**: worktreeパス、ブランチ名、フェーズ名（`requirement`/`analyzed`）等
- **spec-compliance主体の不明確さ**: 完了報告内でspec-complianceの実行責任が不明

これらはすべて `issue-guide` SKILL.md の「完了報告フォーマット」セクション（143-220行目）で定義されている。

## 目標

全4コマンドの完了報告を、CLI引数・内部詳細を含まない自然言語ベースのシンプルな形式に統一する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0006-01 | issue-req完了報告から内部フェーズ名（`Issue状態: {フェーズ}`）を削除し、「要件定義が完了しました」に主眼を移動する |
| REQ-0006-02 | issue-reqのREQ操作表記をCLI的（`CREATE/APPEND/UPDATE`）から自然言語（`新規作成/追記/修正`）に変更する |
| REQ-0006-03 | issue-createのパターンB表記を「REQを作成しました」から「REQをIssue本文に反映しました」に修正する（REQ作成はissue-reqの役割） |
| REQ-0006-04 | issue-createの次ステップからIssue番号引数を削除する |
| REQ-0006-05 | issue-workの4NG分岐を2行の自然言語に集約する（OK→close / NG→フィードバック反映後に再実行） |
| REQ-0006-06 | issue-work完了報告からspec-compliance・issue-updateのCLI引数を削除する |
| REQ-0006-07 | issue-closeの内部詳細（worktreeパス・ブランチ名・Planファイル）を「関連リソースをクリーンアップ」に集約する |
| REQ-0006-08 | issue-closeのパターンB docs表記を簡潔化する（`ドキュメント（REQ・specs）を更新済み`） |

## 非機能要件

- **後方互換性**: issue-update・issue-nextの完了報告フォーマットは変更しない
- **参照整合性**: 「レビューNG時の対応フロー」セクション（224-239行目）と「issue-next推論ルール」セクション（254-264行目）は変更しない
- **コマンドmd参照**: 各コマンドmdファイルが `issue-guide の完了報告フォーマット` を参照している構造は維持する

## スコープ

- **対象**: `.opencode/skills/issue-guide/SKILL.md` の「完了報告フォーマット」セクション（143-220行目）
- **対象外**:
  - レビューNG時の対応フローセクション
  - issue-next推論ルールセクション
  - issue-update完了報告フォーマット
  - 各コマンドmdファイル（フォーマットをインラインしていない限り変更不要）
  - ソースコード・設定ファイル
