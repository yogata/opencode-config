## 背景・課題

`skill-authoring-best-practices` スキルが提供する品質評価基準（5軸 × 4チェック）に基づき、全19スキルを評価した結果、以下の課題が判明した:

1. **Trigger Precision不足**: 7スキルが description に USE FOR / DO NOT FOR パターンを含んでおらず、100+スキルからの選択精度が低い
2. **空description**: `skill-authoring-best-practices` 自体の description が空
3. **Completeness不足**: 2スキル（obsidian系）のSKILL.mdが最小限すぎ、目次として自己完足していない
4. **言語不統一**: 英語description（12スキル）と日本語description（7スキル）が混在

## 目標

全19スキルの description を英語3人称 + USE FOR / DO NOT FOR パターンに統一し、skill-authoring-best-practices の品質基準を全スキルで満たす。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0024-01 | 7スキルの description を英語3人称 + USE FOR / DO NOT FOR パターンに更新する |
| REQ-0024-02 | skill-authoring-best-practices の空 description に適切な英語 description + トリガーを設定する |
| REQ-0024-03 | obsidian-vault-review と obsidian-note-workflow の SKILL.md 本文を補強し、目次として自己完結させる |

### REQ-0024-02: 空description補完

`skill-authoring-best-practices`:
- 現在: description が空
- 変更後例: `Provides quality criteria and best practices for authoring OpenCode SKILL.md files. USE FOR: creating new skills, improving existing skills, reviewing skill quality, or designing skill structure. DO NOT USE FOR: creating command definitions, general coding tasks, or simple documentation fixes.`

### REQ-0024-03: SKILL.md Completeness改善

**obsidian-vault-review** (33行):
- 各レビュー種別（日次/週次/月次/四半期/年次）の概要をSKILL.md内に1-2行ずつ記載
- references/ への導線を明確化

**obsidian-note-workflow** (34行):
- 各ステージ（収集/整理/定着/アーカイブ）の説明をSKILL.md内に1-2行ずつ記載
- Common Mistakesに具体例を追加
