> **⚠️ 非推奨**: この要件ファイルはREQ-0027（要件管理体制の再構築）により廃止されました。新たな領域別要件ファイルに統合されています。

## 背景・課題

epic-status-tracker のステータス表記に不整合がある。進行中・完了・対処不要は「アイコン＋テキスト」（例: `🔄 進行中`）だが、未着手は `☐` のアイコンのみ。他ステータスと形式が異なり、可読性・一貫性に欠ける。

特に新4列形式（`# / Issue / ステータス / 内容`）で `☐` 単独となっており、旧4列形式の `☐ 未着手` とも乖離している。

## 目標

全ステータスを「アイコン＋テキスト」形式に統一し、未着手を `☐ 未着手` に変更する。新4列形式の正規表現パターンも併せて更新する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0026-01 | ステータス値定義テーブルの `☐` を `☐ 未着手` に変更する |
| REQ-0026-02 | 新4列形式の例の `☐` を `☐ 未着手` に変更する |
| REQ-0026-03 | 新4列形式の正規表現（未着手→進行中）を `☐` から `☐ 未着手` に更新する |
| REQ-0026-04 | 新4列形式の正規表現（未着手/進行中→完了）の `☐` を `☐ 未着手` に更新する |
| REQ-0026-05 | べき等性確認パターンの新4列形式で `☐ 未着手` を正しく扱えるよう更新する |

## 非機能要件

特になし（スキルファイル内のテキスト・正規表現変更のみ）

## スコープ

- **対象**: `.opencode/skills/epic-status-tracker/SKILL.md` のステータス表記・正規表現パターン
- **対象外**: 旧4列形式（既に `☐ 未着手` を使用しているため変更不要）、他スキルファイル、コマンド定義ファイル
