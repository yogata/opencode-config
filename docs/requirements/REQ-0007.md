## 背景・課題

- **テンプレート参照の不備**: issue-req, issue-create, issue-update, issue-close の4コマンドがテンプレートファイルを参照する際、バッククォート装飾のみの相対パス（例: `` `templates/doc_requirement.md` ``）を使用している。これはOpenCodeの `@` インライン展開機能を利用しておらず、実行時にテンプレート内容が読み込まれない
- **存在しないスキル参照**: issue-req.md と issue-update.md の `load_skills` に `req-file-manager` を指定しているが、このスキルが `.opencode/skills/` に存在しない。REQ番号採番やファイル操作モード（CREATE/APPEND/UPDATE）の定義が不在

## 目標

- 4コマンドのテンプレート参照を `@` 構文に修正し、実行時にテンプレート内容が確実にインライン展開されるようにする
- `req-file-manager` スキルを新規作成し、REQ番号採番ルールとファイル操作規約を宣言的に定義する

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0007-01 | req-file-manager スキルの新規作成（宣言的知識ベース、REQ番号採番ルール・ファイル操作モード・判定基準・ファイル配置規約・テンプレート参照を定義） |
| REQ-0007-02 | issue-req.md のテンプレート参照を `@.opencode/commands/issue/templates/doc_requirement.md` 構文に修正 |
| REQ-0007-03 | issue-create.md の4テンプレート参照を `@` 構文に修正 |
| REQ-0007-04 | issue-update.md のテンプレート参照を `@.opencode/commands/issue/templates/issue_comment_update.md` 構文に修正 |
| REQ-0007-05 | issue-close.md の両テンプレート参照を `@` 構文に修正 |

## 非機能要件

- req-file-manager/SKILL.md は100行以内
- 既存スキル（issue-guide, req-analysis等）のロジックを変更しない
- テンプレートファイル（14ファイル）の内容を変更しない

## スコープ

- **対象外**: issue-work.md, issue-next.md, README.md（テンプレート参照なし）、テンプレートファイル自体（14ファイル）、既存スキル、commands-creator スキル
