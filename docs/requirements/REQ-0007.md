## 目的

開発過程で得られた学びを3層パイプライン（キャプチャ→分析→昇華）で蓄積・活用する仕組みの要件を定義する。暗黙知の形式知化と再利用性の向上を目的とする。

## 要件

| ID | 要件 |
|---|---|
| REQ-0007-001 | 学びの検知・抽出・提案はエージェントが主体となって行い、ユーザーは承認または却下のみを行うものとする。エージェントが「学びはありましたか？」と問いかけることを禁ずる（SHALL） |
| REQ-0007-002 | キャプチャ層（tips-add）は学びを`docs/tips/inbox.md`に追記し、分析層（tips-refactor）はinboxをセマンティック分析して`evaluation-report.md`を生成しarchiveへ移動するものとする（SHALL） |
| REQ-0007-003 | 昇華層（tips-elevate）は評価レポートから昇華判定を行い、staging領域にスタブを生成するものとする（SHALL） |
| REQ-0007-004 | inbox.mdのエントリ数が15件以上に達した場合、tips-refactorの実行を推奨するものとする（SHOULD） |
| REQ-0007-005 | 学びのフォーマットはタイトル・状況・原因・解決策・教訓・タグの6項目で構成するものとする。タグは人間の可読性用であり、セマンティック分析は内容ベースで行う（SHALL） |
| REQ-0007-006 | issue-close完了時に学びの検知を実行し、ユーザーの承認を経てinbox.mdに追加するものとする（SHALL） |

## 適用範囲

- **対象**: 3層パイプライン（キャプチャ・分析・昇華）のデータフロー、学びのフォーマット、エージェント主体の検知プロトコル、inbox閾値判定
- **対象外**: セマンティック分析のアルゴリズム、staging領域の具体的な出力形式
