## 目的

SisyphusフレームワークにおけるPlan管理の規則を定義する。Planの命名規則・アーカイブ手順・ディレクトリ構造の整合性を維持し、開発作業の再開可能性とトレーサビリティを担保するための要件である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0033-001 | `.sisyphus/`配下の7カテゴリ（plans・drafts・evidence・execution・notepads・tasks・reports）のファイル・ディレクトリ命名はplan名を基準とするものとする（SHALL） |
| REQ-0033-002 | notepadsカテゴリは完全一致マッチングのみ対応し、plan名のサフィックスを含めてディレクトリ名とするものとする。他カテゴリはプレフィクスマッチングを許容する（SHALL） |
| REQ-0033-003 | 完了したPlanに関連する全ファイル（plans・drafts・evidence・execution・notepads・tasks・reports）を`.sisyphus/archives/<plan_name>/`に移行するものとする（SHALL） |
| REQ-0033-004 | notepadsカテゴリのアーカイブ時は、ディレクトリ内のコンテンツを直接アーカイブ先に移動するものとする。隠しファイル（`.`で始まるファイル）はスキップする（SHALL） |
| REQ-0033-005 | 空のnotepadディレクトリは作成してはならない（SHALL） |
| REQ-0033-006 | AGENTS.mdにSisyphus命名規則を記載し、エージェントが命名規則を参照できるものとする（SHALL） |

## 適用範囲

- **対象**: `.sisyphus/`のディレクトリ構造と命名規則、Plan完了時のアーカイブ手順、AGENTS.mdの命名規則記載
- **対象外**: Planの内容フォーマット、実行エージェントの選択、Boulder（作業単位）の内部構造
