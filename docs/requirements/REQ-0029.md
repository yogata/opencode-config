## 目的

8つのissue-*コマンドが従うべき入出力契約・実行順序・データフローを定義する。コマンド間の結合度を明示し、ワークフロー全体の整合性と拡張性を担保するための通信規約である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0029-001 | issue-reqはセッション会話を入力とし、要件docを出力するものとする。対話系コマンドとしてprometheusエージェントを使用する（SHALL） |
| REQ-0029-002 | issue-save-reqは`.sisyphus/drafts/`のドラフトを入力とし、REQファイル・ADRファイル・インデックス更新を出力するものとする（SHALL） |
| REQ-0029-003 | issue-createは要件docおよびspecs・ADRを読み取り、GitHub Issueを出力するものとする。Epic規模の場合はEpic + 子Issueを一括作成する（SHALL） |
| REQ-0029-004 | issue-workはGitHub Issueおよびspecsを読み取り、実装済みブランチ・PRを出力するものとする。3フェーズ構成（準備・実装・提出）でべき等な再開ポイントを提供する（SHALL） |
| REQ-0029-005 | issue-closeはPRとIssueを入力とし、マージ完了・記録追記・ブランチ削除を実行するものとする。REQステータス更新および学び抽出を含む（SHALL） |
| REQ-0029-006 | issue-nextは現在のセッション状態を分析し、次に実行すべきコマンドを推論して提示するものとする（SHALL） |
| REQ-0029-007 | issue-backlogはユーザー指定の期間内にクローズされたIssue/PRから残課題を抽出し、Epic + 子Issueとして作成するものとする（SHALL） |
| REQ-0029-008 | issue-updateはIssue本文の更新・コメント追加に加え、REQファイルのAPPEND/UPDATE操作をサポートするものとする（SHALL） |

## 適用範囲

- **対象**: 8つのissue-*コマンドの入出力契約、実行順序、コマンド間データフロー、エージェント指定
- **対象外**: コマンド内の詳細手順、並列実行の依存分析、テンプレートの内容、gh CLIの安全性手順
