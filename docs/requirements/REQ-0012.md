## 目的

本リポジトリで使用する自然言語を原則として日本語に統一するルールを定義する。ユーザーおよびエージェントが読む説明文の言語を統一し、一貫性と可読性を担保するための要件である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0012-001 | README、docs、commands（説明文）、skills、テンプレート、Issue本文、PR本文、完了報告、REQファイル、ADRファイル、specs、AGENTS.md、コミットメッセージのdescription等、ユーザーまたはエージェントが読む説明文は、原則として日本語で記述するものとする（SHALL） |
| REQ-0012-002 | コマンド名、ファイル名、ディレクトリ名、ラベル名、Conventional Commitsのtype（feat/fix/refactor等）、技術仕様名、固有名詞、frontmatterキー、ツールや仕様上の予約語は英語表記を許容するものとする（SHALL） |
| REQ-0012-003 | 英語表記を見出しまたは識別子として維持する場合でも、その説明本文は原則として日本語で記述するものとする（SHALL） |

## 適用範囲

- **対象**: 自然言語の使用言語ルール、英語許容例外の定義、見出し英語時の本文ルール
- **対象外**: コード内識別子命名（変数名・関数名等）、外部ライブラリのAPI名
