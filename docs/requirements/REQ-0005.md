## 目的

大規模機能追加におけるEpic Issueと子Issueの親子関係管理・ステータス追跡・自動クローズの規則を定義する。Epic進捗の可視化と確実な完了処理を担保するための要件である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0005-001 | Epic Issueは子Issue本文に`Parent: #{N}`パターンで親Issue番号を記載し、親子関係を明示するものとする（SHALL） |
| REQ-0005-002 | 親Epic本文にステータス追跡テーブル（4列形式）を配置し、各子Issueの進捗を`☐ 未着手`・`🔄 進行中`・`✅ 完了`・`❌ 対処不要`の4値で管理するものとする（SHALL） |
| REQ-0005-003 | 子Issueの作業開始時（issue-work Phase A）にEpicステータスを`🔄 進行中`に更新し、完了時（issue-close Step 8）に`✅ 完了`に更新するものとする。更新はべき等であること（SHALL） |
| REQ-0005-004 | 全子Issueが終了状態（`✅ 完了`または`❌ 対処不要`）に到達した場合、親Epicを自動クローズするものとする。残存する未完了子Issueがある場合はスキップする（SHALL） |
| REQ-0005-005 | `❌ 対処不要`は手動設定のみの終了状態とし、自動設定してはならない（SHALL） |
| REQ-0005-006 | ステータス更新時は新旧4列形式の両方をサポートし、正規表現パターンで該当行を特定・置換するものとする（SHALL） |

## 適用範囲

- **対象**: Epic + 子Issueの親子関係定義、ステータス追跡テーブルの形式と更新プロトコル、Epic自動クローズ判定、ステータス値の定義
- **対象外**: Issue作成手順（issue-createの管轄）、並列実行のWaveスケジューリング、worktree管理
