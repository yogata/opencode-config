# issue-*コマンドのサブエージェント最終出力のverbatim出力

## 背景・課題

issue-*コマンドは`call_omo_agent`、バックグラウンドタスク、`/start-work`、`@plan`等のサブエージェントを利用する。サブエージェントは`load_skills`経由で`issue-guide`を読み込んでおり、完了報告フォーマット（表・ハイライト・コードブロック等のMarkdown表現を含む）を把握した上で出力を生成している。

しかし、親エージェントがサブエージェントの出力を受け取った後、再フォーマット・要約・再構成を行うことで、以下の問題が発生している：

1. **表の崩壊**: サブエージェントが生成したMarkdownテーブルが親エージェントの再フォーマットにより崩れる
2. **ハイライトの消失**: 強調表示（`**太字**`等）が要約時に失われる
3. **情報の欠落**: サブエージェントが意図的に含めた詳細情報が、親エージェントの要約により削除される

## 目標

サブエージェントの最終出力（完了報告・実装サマリ・乖離検出レポート等）を、親エージェントがそのまま（verbatim）ユーザーに表示する。再フォーマット・要約・再構成を禁止する。

## 方向性

issue-guideスキルに「サブエージェント出力ポリシー」を追加し、各コマンドのGuardrailsにverbatim出力の義務を明記する。サブエージェントは既にissue-guideの完了報告フォーマットを把握しているため、親エージェントによるフォーマット適用は不要・有害である。

## 機能要件

- [ ] issue-guideにサブエージェント出力ポリシーを追加
  - **Given**: issue-guideスキルの完了報告フォーマットセクションが存在する
  - **When**: 「サブエージェント出力ポリシー」セクションを追加し、サブエージェントの最終出力のverbatim出力を義務付ける
  - **Then**: ポリシーに「サブエージェントの最終出力は再フォーマット・要約・再構成せず、そのままユーザーに表示すること」が明記される

- [ ] 各コマンドのGuardrailsにverbatim出力ガードレールを追加
  - **Given**: issue-req, issue-create, issue-work, issue-update, issue-close, issue-nextの各コマンドファイルが存在する
  - **When**: 各コマンドのGuardrailsセクションに「サブエージェントの最終出力はverbatimで出力する（再フォーマット禁止）」を追加する
  - **Then**: 全issue-*コマンドでサブエージェント出力のverbatim扱いがガードレールとして明示される

- [ ] verbatim出力の対象を明確化
  - **Given**: サブエージェントが生成する出力には複数の種類がある
  - **When**: ポリシーで「最終出力」の定義を明記する（完了報告、実装サマリ、乖離検出レポート、review-work結果等）
  - **Then**: 開発者がどの出力をverbatim扱いすべきか判断できる

**品質基準**: 各要件は 測定可能・一意・実装可能 であること。

## 非機能要件

- 既存の完了報告フォーマット（issue-guide定義）は維持する。サブエージェントがこのフォーマットで出力することを前提とする
- 親エージェントの完了報告フォーマット（✅、🎉等）は、サブエージェントからの出力がない場合のみ親エージェント自身が出力する

- 新しいコマンドやスキルの追加
- `call_omo_agent`の仕様変更
