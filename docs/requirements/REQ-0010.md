## 背景・課題

issue関連6コマンド・6スキル間の連携に5つのギャップが存在し、Anchored Developmentワークフローの完成度が80%に留まっている。

**特定されたギャップ**:

| # | ギャップ | 影響 |
|---|---|---|
| G1 | 5コマンドでload_skills計11箇所が欠落 | コマンド実行時に必要スキルがロードされず、要件参照・ADR確認等が機能しない |
| G2 | 全6スキルにSee Alsoセクション不在 | スキル間の発見性が極めて低い（req-file-managerは完全孤立） |
| G3 | issue-guideハブの参照マトリクスに3スキル欠落 | adr-guidelines, adr-file-manager, req-file-managerがハブから参照されていない |
| G4 | スパイク/研究ワークフローのテンプレート不在 | 技術調査Issueが既存テンプレート（feature/bug）に強制適合される |
| G5 | Issueテンプレートにテスト戦略セクション不在 | TDD必須のプロジェクト方針とテンプレートが乖離 |
| G6 | README.mdが実際のload_skillsと不整合 | スキル一覧・コマンド参照が不正確 |

## 目標

全ギャップを解消し、Anchored Developmentワークフローの完成度を95%に引き上げる。具体的には、全コマンドのload_skills完備・全スキルのSee Alsoセクションによる相互参照確立・テンプレートカバレッジ向上を実現する。

## 非機能要件

- **非破壊性**: 既存セクションの削除・変更は禁止（追記のみ許可）
- **対称性**: See Alsoの双方向参照に過不足がないこと（A→Bの参照があればB→Aの参照も存在すること）
- **整合性**: README.md記載内容と実際のファイル内容（load_skills、スキル一覧、テンプレート一覧）が完全に一致すること
- **パターン適合**: 新規テンプレートは既存テンプレートと同じfrontmatter形式・セクション構成スタイルに従うこと

## スコープ

- **対象**: `.opencode/commands/issue/`配下のコマンドファイル・テンプレート、`.opencode/skills/`配下の6スキル（adr-file-manager, adr-guidelines, req-file-manager, req-analysis, spec-compliance, issue-guide）、issue/README.md
- **対象外**: 新規コマンド/スキルの作成、ソースコード変更、ホットフィックス/リリース専用テンプレート、既存セクションの削除・変更
