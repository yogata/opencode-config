## 目的

OpenCode SKILL.mdファイルの品質基準とオーサリング規約を定義する。スキルの選択精度・保守性・トークン効率を担保し、一貫した品質のスキル群を維持するための要件である。

## 要件

| ID | 要件 |
|---|---|
| REQ-0035-001 | スキルのfrontmatterには`name`と`description`のみを含めるものとする。拡張フィールドの追加はしてはならない（SHALL） |
| REQ-0035-002 | descriptionにはトリガー条件を`USE FOR:`と`DO NOT USE FOR:`（または`WHEN:`）の形式で埋め込み、3人称で記述するものとする（SHALL） |
| REQ-0035-003 | SKILL.mdの行数は500行以下とし、推定トークン数は複雑度に応じた予算内（simple: <2K・moderate: <4K・detailed: <5K）に収めるべきである（SHOULD） |
| REQ-0035-004 | Progressive Disclosureパターンを適用し、SKILL.mdを目次として機能させ、詳細は参照ファイル（1階層まで）に配置するものとする。深いネストは禁止する（SHALL） |
| REQ-0035-005 | スキル品質を5つの評価軸（明確性・完全性・トリガー精度・スコープ適合性・アンチパターン検出）で評価するものとする（SHALL） |
| REQ-0035-006 | スキル開発は反復アプローチ（スキルなしでタスク完了→パターン特定→スキル作成→テスト→改善）で進めるべきである（SHOULD） |
| REQ-0035-007 | namingは小文字・数字・ハイフンのみでkebab-caseとし、動名詞形（gerund form）を推奨するものとする（SHALL） |

## 適用範囲

- **対象**: SKILL.mdの構造規約、トリガー設計、トークン予算管理、品質評価軸、Progressive Disclosureパターン、命名規則
- **対象外**: コマンド定義の規約（commands-creatorの管轄）、スキル読み込みの仕組み（OpenCodeランタイムの管轄）
