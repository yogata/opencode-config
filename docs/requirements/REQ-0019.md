## 背景・課題

`issue-backlog` コマンドはクローズ済みissue/PRから残課題を抽出し、Epic + 子Issueを作成する。しかし、抽出後に抽出元issue/PRへのマーキングがないため、同じ期間を再実行すると同一残課題が再抽出され、重複するEpic + 子Issueが作成されてしまう。

現状のGuardrailsには「同一期間への再実行時は新規Epicを作成」とあるが、これは重複を許容する記述であり、根本的な解決になっていない。

## 目標

抽出元issue/PRに「バックログ抽出済み」を示すマーカーコメントを投稿し、再実行時に該当issue/PRを自動的にスキップすることで、重複抽出を防止する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0019-01 | Epic + 子Issue作成完了後、抽出元の各issue/PRに「バックログ抽出済み」コメントを投稿する |
| REQ-0019-02 | コメントには可視テキスト `backlog-extracted` を含め、Epic Issueへのリンク・子Issueへのリンク・抽出日を記載する |
| REQ-0019-03 | データ取得後・抽出処理前に、各issue/PRのコメントに `backlog-extracted` キーワードが含まれているか確認し、含まれている場合は当該issue/PRを抽出対象からスキップする |
| REQ-0019-04 | スキップされたissue/PRはレポートに表示せず、静かに除外する |

## 非機能要件

- **可用性**: コメント投稿に失敗した場合でも、Epic + 子Issue作成自体は成功扱いとする。失敗したissue/PR番号を完了報告に含める
- **保守性**: マーカーキーワード `backlog-extracted` は1箇所で定義し、検索・投稿の両方で参照する

## スコープ

- **対象**: `issue-backlog` コマンド定義ファイル（`.opencode/commands/issue/issue-backlog.md`）の Steps・Guardrails の変更
- **対象外**: 他の issue-* コマンドの変更、テンプレートの新規作成
