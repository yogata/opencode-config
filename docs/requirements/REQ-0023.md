## 背景・課題

epic-status-tracker SKILL.md のステータス追跡テーブルフォーマットに、以下の改善が必要：

1. **連番がない**: #695（残課題Epic）のようなセクション-連番形式（`1-1`, `1-2`...）がないため、項目の特定が困難
2. **ステータス値が限定的**: `☐ 未着手` / `🔄 進行中` / `✅ 完了` の3値のみで、「対処不要」を表現できない
3. **完了にPR情報がない**: `✅ 完了` のみでPR番号が含まれず、対応PRへのトレーサビリティがない
4. **SKILL.mdに追加禁止制約**: `新しいステータス値を追加しないこと。` という制約があり、拡張を阻害している

## 目標

epic-status-tracker のテーブルフォーマットを #695 の形式に近づけ、可読性・トレーサビリティ・拡張性を向上させる。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0023-01 | テーブルに `#` 列を追加し、セクション-連番形式（`1-1`, `1-2`, `2-1`...）で採番する。セクション番号はカテゴリグループに対応し、連番の末尾がセクション内件数と一致する |
| REQ-0023-02 | 未着手の表現を `☐ 未着手` から `☐`（空のチェックボックスのみ）に変更する |
| REQ-0023-03 | `❌ 対処不要` を新しいステータス値として追加する。設定は手動のみ（issue-work/issue-close による自動遷移なし）。端状態として扱い、自動上書きしない |
| REQ-0023-04 | 完了ステータスにPR番号を含める: `✅ 完了 ([PR#N](URL))` 形式。issue-close 実行時に自動設定する |
| REQ-0023-05 | Epic自動クローズ判定において、`❌ 対処不要` を `✅ 完了` と同等の「終了」状態として扱う |
| REQ-0023-06 | SKILL.md の `新しいステータス値を追加しないこと。` 制約を削除し、将来のステータス追加を可能にする |

## 非機能要件

- **後方互換性**: 既存の正規表現パターンを新フォーマットに更新し、移行期間中の不一致を防止する
- **べき等性**: 既存のべき等性確認ロジック（既に目標ステータスならスキップ）は維持する

## スコープ

- **対象**: epic-status-tracker SKILL.md のテーブルフォーマット定義、ステータス値定義、正規表現パターン、および依存ファイル（issue_desc_epic.md テンプレート、issue-work.md、issue-close.md、issue-guide-phases/SKILL.md）
- **対象外**: issue-backlog コマンド（別フォーマットを生成）、他プロジェクトへの横展開

## ステータス値定義（変更後）

| 値 | 意味 | 設定タイミング | 終了状態 |
|---|---|---|---|
| `☐` | 子Issue未着手 | Epic作成時（初期値） | いいえ |
| `🔄 進行中` | 子Issue作業中 | `issue-work` Phase A | いいえ |
| `✅ 完了 ([PR#N](URL))` | 子Issue完了 | `issue-close` Step 8 | はい |
| `❌ 対処不要` | 対処不要（手動設定） | ユーザー手動 | はい |
