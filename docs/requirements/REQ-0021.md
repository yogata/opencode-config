## 背景・課題

現在の `skill-authoring-best-practices/SKILL.md`（427行）は、スキル作成の基本的なガイドラインを提供しているが、**品質の客観的評価・改善**において以下の課題がある：

1. **定量的指標の欠如**: 「500行以内」という行数基準のみで、トークン予算の概念がない
2. **トリガー定義の浅さ**: description書き方ルールで「Use when ...」を推奨しているが、ネガティブトリガー（使わない場面）の指摘がない。agentskills.io のデファクト慣習（`USE FOR` / `DO NOT USE FOR`）にも未対応
3. **品質評価の薄さ**: 9項目のチェックリストのみで、評価軸が構造化されていない
4. **複雑度分類の不在**: すべてのスキルを同じ基準で扱い、シンプルなスキルと複雑なスキルの区別がない

Microsoft Waza（AIエージェントスキル評価ツール）の分析から、これらの課題に対応する5つの有用概念を特定した。Wazaツール自体は導入しないが、概念のみをマージする。

## 目標

`skill-authoring-best-practices` を全面的に再構成し、Wazaの品質概念5つを取り込んで、**スキル品質の客観的評価・改善基準**を確立する。

## 機能要件

| ID | 要件 |
|---|---|
| REQ-0021-01 | 全体構成を7セクション（目的別3層構造）に再構成する |
| REQ-0021-02 | セクション1「設計原則」に Token Budget Awareness を新設し、agentskills.io 仕様に基づく定量的指標（500行以内 / <5000トークン）を定義する |
| REQ-0021-03 | セクション2「スキル仕様」に Trigger Design ガイドラインを新設し、description 内に `USE FOR:` / `DO NOT USE FOR:` を記述する慣習を agentskills.io デファクトに沿って推奨する |
| REQ-0021-04 | セクション3「構造設計」に Complexity Classification（simple / moderate / detailed）を新設し、複雑度に応じた構造・予算ガイダンスを定義する |
| REQ-0021-05 | セクション4「品質評価基準」を新設し、Wazaの5軸（Clarity / Completeness / Trigger Precision / Scope Coverage / Anti-Pattern Detection）を OpenCode 向けに適応して定義する。自己チェック・レビュー兼用とする |
| REQ-0021-06 | セクション5「レビュープロトコル」を新設し、Frontmatter / Budget / Structure / Advisory の4観点でチェック手順を定義する |
| REQ-0021-07 | 既存セクション（Workflows, Anti-Patterns, Checklist 等）の内容を再構成後のセクションに再配置し、内容の損失がないこと |
| REQ-0021-08 | 再構成後の総行数を500行以内に収める（agentskills.io ガイダンス準拠） |

## 非機能要件

- **後方互換性**: 既存スキルへの影響なし（skill-authoring-best-practices は参照されるのみで、他スキルの構造に依存しない）
- **保守性**: セクション構造を目的別に整理し、将来の追加・変更が容易な構造とする
- **記述言語**: 既存と同じく日本語で記述する

## スコープ

- **対象**: `.opencode/skills/skill-authoring-best-practices/SKILL.md` の再構成
- **対象外**:
  - 他スキルの SKILL.md の変更
  - Waza ツールの導入
  - 新規スキルの作成
  - agentskills.io 仕様への full準拠（frontmatterの `license`, `metadata`, `allowed-tools` 等の追加は不要）
  - 自動テストの追加（Markdownファイルのみ）
